# 对话式知识库接入设想

目标是把自然语言请求收束成 add、append、update-meta、search、context 等动作，并保持 Markdown 为唯一真源。

## 接入建议

优先让对话层负责意图识别，再把结构化参数交给本地脚本执行，这样更容易控制 token 和修改边界。
