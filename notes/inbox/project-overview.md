# 项目总览

这个项目的目标不是单纯写博客，而是做一个以 **Markdown 为唯一真源** 的个人知识系统。

它主要解决四个问题：

## 1. 让分散的笔记变成可持续生长的知识库

过去的笔记可能散落在不同目录、不同文档里，查找和复用成本很高。  
这个项目把所有知识内容统一收进 `notes/*.md`，再用本地脚本生成统一索引、目录和导航。

## 2. 让站点和 AI 使用同一份知识数据

以前最容易出问题的地方，是前端有一套数据来源、后端又有另一套接口逻辑。  
这次重构后的核心原则是：

- 前端目录来自统一的知识索引
- 后端搜索来自统一的知识索引
- AI 问答也先基于统一的知识索引检索

这样前后端不会再各自维护一套难以对齐的状态。

## 3. 让 AI 先检索、再回答

这个项目并不希望 AI 直接“自由发挥”，而是要求它先找到相关笔记，再基于命中内容做整理。  
如果没有命中结果，就应该明确说不知道，而不是编造答案。

## 4. 让知识系统足够简单，方便后续继续扩展

当前版本只保留真正必要的部分：

- Markdown 知识库
- 本地知识索引
- 站点首页和知识目录
- 检索优先的 AI 工作台
- 可选的远程大模型整理能力

后续如果要继续扩展，再往这个基础上增加写入动作、权限管理或网页导入能力会更稳。

## 当前重构后的整体思路

现在的项目结构可以概括成一句话：

> 用 Markdown 存知识，用统一索引驱动站点，用检索优先的 API 和 AI 工作台消费这份知识。

如果你只想快速理解这个项目，优先看这三处：

- 首页：看整体定位
- `/notes/`：看知识内容怎么被组织
- `/ai/`：看 AI 如何围绕知识检索工作
